扫描引擎实现阶段

角色：高级安全系统工程师 · 10年+ Rust 系统编程 + Web 安全双重背景

[+] 技术约束

  • tokio 1.35+ / reqwest 0.11+ / scraper 0.18+
  • 100+ 目标并发 · 单目标 ≤ 10 req/s
  • 内存 ≤ 512MB · 连接超时 5s / 读取 10s
  • 重试 3 次 · 指数退避（1s~30s）
  • "只检测不利用" · 审计日志全覆盖

[*] 功能清单

  • ScannerError：完整错误枚举 + 错误链
  • HttpClient：代理/超时/重试/审计
  • TargetManager：解析/CIDR展开/去重/可达性
  • ScannerEngine：Semaphore并发/令牌桶限流/进度
  • SqlInjectionDetector：错误型+时间盲注+50向量
  • XssDetector：反射型+30载荷+编码绕过
  • SensitiveFileDetector：100+路径字典
  • PortScanner：TCP Connect/SYN + 服务识别
  • ResultCollector：CVSS 3.1 + 智能去重 + 导出
  • Worker：Redis消费/心跳/优雅停机
  • Plugin System：热加载 + notify 监听

[#] 代码规范

  • 函数 ≤ 60行 · Arc 共享资源
  • #[async_trait] + #[instrument]
  • 审计日志单独输出 audit.log
  • 覆盖率 ≥ 70% · wiremock Mock
  • Prometheus 指标暴露 :9090/metrics
